# Checklist Breve de Pruebas y Entrega

## 1) Checklist de pruebas mínimas
- [ ] Ejecuté el flujo principal al menos una vez.
- [ ] Probé una entrada válida.
- [ ] Probé una entrada inválida o caso límite.
- [ ] Registré resultados en `docs/plan_de_pruebas.md`.

## 2) Checklist de documentación
- [ ] Completé `docs/propuesta.md`.
- [ ] Completé `docs/caso_de_uso.md`.
- [ ] Revisé `docs/estructura_repositorio.md` y respeté la estructura.
- [ ] Definí criterios de éxito claros y medibles.

## 3) Checklist de ejecución
- [ ] `scripts/run.sh` funciona en mi entorno.
- [ ] El archivo principal en `src/` coincide con mi lenguaje elegido.
- [ ] El programa muestra mensajes claros en terminal.

## 4) Checklist de entrega final
- [ ] El proyecto mantiene alcance pequeño.
- [ ] No usé frameworks, nube, Docker, APIs externas ni bases de datos.
- [ ] El repositorio está limpio y ordenado.
- [ ] La entrega está lista para revisión en GitHub Classroom.
