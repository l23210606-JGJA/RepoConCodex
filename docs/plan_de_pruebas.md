## 4) Pruebas con errores
Define pruebas con entradas inválidas o incompletas y documenta el comportamiento esperado.

## 5) Pruebas mínimas por lenguaje
- **ARM64 Assembly:** ejecución básica, lectura/escritura mínima, validación de salida esperada.
- **C:** compilación correcta, ejecución con entrada válida e inválida.
- **Python:** ejecución del script principal con caso normal y caso límite.
- **Bash:** permisos de ejecución, manejo de argumentos y mensajes de error claros.

## 6) Criterios para considerar la práctica terminada
- La documentación está completa y coherente.
- Existen casos de prueba mínimos ejecutados.
- El comportamiento principal está validado.
- El repositorio está ordenado y listo para revisión en GitHub Classroom.

> Nota: no se requieren frameworks de testing para esta práctica.
